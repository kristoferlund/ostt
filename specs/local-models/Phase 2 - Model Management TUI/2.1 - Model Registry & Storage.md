# 2.1 — Model Registry & Storage

## Problem

OSTT needs to know which local models are available, which ones are installed on this machine, and which one is active. Available official models come from a GitHub-hosted registry JSON. The registry is not just a URL list: it is the product surface for discovering canonical Whisper models, Swedish/regional fine-tunes, hardware recommendations, language metadata, and community additions.

The GitHub registry tells OSTT what official models exist. The filesystem tells OSTT what model files are installed, their size, and modified/created time. OSTT's existing selected-model mechanism tells OSTT which provider/model is active. Local model state stores custom model entries that are not in the GitHub registry.

## Objective

1. Define the GitHub registry format and local custom model state format (`models.json`).
2. Implement read/write operations for local state.
3. Define the shared registry entry shape used by GitHub registry entries and custom model entries.
5. Implement installed-model discovery by listing the model directory and matching files to registry/custom entries.
6. Implement model activation/deactivation: switching the active model in local state.
7. Implement model deletion: removing model files from disk and clearing active state if needed.

## Specification

### Directory structure

```
~/.local/share/ostt/models/
├── models.json            # Local state: custom model entries only
└── files/
    ├── turbo.bin
    ├── kb-whisper-large.gguf
    └── ...
```

All downloaded model files live in one folder. File size and modified/created time are read from the filesystem.

### Registry entry shape

```rust
use serde::{Serialize, Deserialize};

/// A model that OSTT can present for download or activation.
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct RegistryEntry {
    pub id: String,
    pub name: String,
    pub description: String,
    /// "multi", "sv", "en", etc. Used for browsing and language-aware defaults.
    pub languages: Vec<String>,
    pub size_mb: u32,
    /// Direct model download URL.
    pub url: String,
    pub recommended_hardware: Option<String>,
    pub sha256: Option<String>,
    /// Optional display hint: "default", "swedish", "regional", "experimental".
    pub category: Option<String>,
}
```

The GitHub registry uses this shape. Custom model entries use the same shape. The TUI, auth flow, and recommendation logic consume this metadata directly.

### File naming

Model file names are derived from the registry/custom model ID, not stored separately in local state.

Model IDs must be safe to use as file names on all supported platforms. They may contain lowercase ASCII letters, numbers, dots, dashes, and underscores only:

```text
^[a-z0-9._-]+$
```

OSTT model IDs use safe short IDs. The registry entry URL points to the actual model file.

```rust
pub fn model_filename(id: &str, url: &str) -> String {
    let ext = extension_from_url(url).unwrap_or("bin");
    format!("{id}.{ext}")
}
```

Examples:
- `turbo` → `turbo.bin`
- `kb-whisper-large` → `kb-whisper-large.bin` or `.gguf`, depending on download URL extension

Remote registry CI must verify:
- `id` is unique
- `id` matches `^[a-z0-9._-]+$`
- derived `model_filename(id, url)` is unique
- required fields are present

For custom URL/model-ID downloads, ask the user for at least an `id` and `name` after resolving metadata. Derive the local filename from that custom ID. If the derived filename already exists, warn the user and require a different ID or explicit replacement.

### Local state file: `models.json`

`models.json` stores local model state that cannot be derived from the GitHub registry, filesystem, or existing selected-model state.

```rust
/// Top-level local state.
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct LocalModelState {
    /// Schema version (for forward compatibility)
    pub version: u32,
    /// Custom models that are not present in the GitHub registry.
    pub custom_models: Vec<RegistryEntry>,
}
```

Registry models do not need to be written into `models.json` when downloaded. Installed status is inferred by checking whether `files/model_filename(entry.id, entry.url)` exists.

The active local model is stored using OSTT's existing selected-model mechanism (`~/.local/share/ostt/model` today). Local support should reuse/extend that mechanism rather than adding a second active-model field. Since local model IDs are data-driven and may not parse as `TranscriptionModel`, the selected-model mechanism should be extended to preserve the selected provider ID alongside the model ID.

### Serialization helpers

In `src/transcription/local_models.rs`:

```rust
use std::fs;
use std::path::PathBuf;

fn state_path() -> PathBuf {
    models_dir().join("models.json")
}

/// Load local model state. Returns default state if file doesn't exist.
pub fn load_state() -> LocalModelState {
    let path = state_path();
    if path.exists() {
        let content = fs::read_to_string(&path).unwrap_or_default();
        serde_json::from_str(&content).unwrap_or_default()
    } else {
        LocalModelState::default()
    }
}

/// Save local model state to disk.
pub fn save_state(state: &LocalModelState) -> anyhow::Result<()> {
    let path = state_path();
    fs::create_dir_all(path.parent().unwrap())?;
    let content = serde_json::to_string_pretty(state)?;
    fs::write(&path, content)?;
    Ok(())
}
```

Implement `Default` for `LocalModelState`:

```rust
impl Default for LocalModelState {
    fn default() -> Self {
        Self {
            version: 1,
            custom_models: Vec::new(),
        }
    }
}
```

### Initial GitHub registry contents

The GitHub registry includes the canonical Whisper.cpp models and selected regional/community models. Initial entries should include at least:
- `tiny`, `base`, `small`, `medium`, `large-v3`, `turbo`
- `kb-whisper-large` — Swedish, critical for OSTT's likely audience
- `nb-whisper-large` — Norwegian
- Distil-Whisper GGUF variants where compatible with whisper.cpp

Use the actual upstream URL and preserve enough extension/format information to save a loadable file. Do not assume all models end in `.gguf`; whisper.cpp-hosted Whisper assets may still use `ggml-*.bin`. The compatibility check is "can whisper.cpp load it", not filename extension alone.

### Installed model discovery

```rust
pub struct InstalledModelView {
    pub entry: RegistryEntry,
    pub path: PathBuf,
    pub size_bytes: u64,
    pub modified_at: Option<SystemTime>,
    pub is_active: bool,
}

pub fn installed_models(
    registry: &[RegistryEntry],
    state: &LocalModelState,
    selected_model: Option<&SelectedModel>,
) -> Vec<InstalledModelView> {
    let entries = registry.iter().chain(state.custom_models.iter());
    entries
        .filter_map(|entry| {
            let path = model_files_dir().join(model_filename(&entry.id, &entry.url));
            let metadata = fs::metadata(&path).ok()?;
            Some(InstalledModelView {
                entry: entry.clone(),
                path,
                size_bytes: metadata.len(),
                modified_at: metadata.modified().ok(),
                is_active: selected_model
                    .map(|selected| selected.provider_id == "local" && selected.model_id == entry.id)
                    .unwrap_or(false),
            })
        })
        .collect()
}
```

Downloaded status in UI is computed the same way: derive path from registry/custom entry and check if the file exists.

### Activation

```rust
/// Set a model as the active transcription model.
pub fn activate_model(model_id: &str) -> anyhow::Result<()> {
    let state = load_state();
    let registry = fetch_or_load_registry()?;
    let entry = find_model_entry(model_id, &registry, &state)?;
    let path = model_files_dir().join(model_filename(&entry.id, &entry.url));
    if !path.exists() {
        anyhow::bail!("Model '{model_id}' is not downloaded. Run `ostt model` and open local model management first.");
    }
    save_selected_model("local", model_id)
}

/// Deactivate the current model (revert to none).
pub fn deactivate_model() -> anyhow::Result<()> {
    clear_selected_model()
}
```

### Deletion

```rust
/// Remove a downloaded model from disk and state.
pub fn delete_model(model_id: &str) -> anyhow::Result<()> {
    let state = load_state();
    let registry = fetch_or_load_registry()?;
    let entry = find_model_entry(model_id, &registry, &state)?;
    let file_path = model_files_dir().join(model_filename(&entry.id, &entry.url));

    // Delete the file
    if file_path.exists() {
        fs::remove_file(&file_path)?;
    }

    // If this was the active model, clear active
    if get_selected_model()?.as_ref().is_some_and(|selected| {
        selected.provider_id == "local" && selected.model_id == model_id
    }) {
        clear_selected_model()?;
    }

    // Keep custom model metadata so a deleted custom model can still be re-downloaded/inspected.
    Ok(())
}
```

## Acceptance Criteria

### State persistence
- [ ] After calling `save_state()`, `models.json` exists at the correct path
- [ ] After calling `load_state()`, returns the previously saved state
- [ ] If `models.json` doesn't exist, `load_state()` returns default (empty custom model list)
- [ ] Corrupted `models.json` returns default (resilient)
- [ ] `save_state()` creates parent directories if they don't exist
- [ ] Downloaded GitHub registry models are not duplicated into `models.json`
- [ ] Installed status is inferred by checking files in `models/files/`
- [ ] Active model selection is not stored in `models.json`

### Activation
- [ ] `activate_model("turbo")` saves provider/model through OSTT's selected-model mechanism
- [ ] `activate_model("nonexistent")` returns an error mentioning the model isn't downloaded
- [ ] `activate_model("turbo")` verifies the derived model file exists before activation
- [ ] `deactivate_model()` clears OSTT's selected-model state
- [ ] Activation persists across selected-model load/save cycles

### Deletion
- [ ] `delete_model("turbo")` removes the derived model file
- [ ] `delete_model("turbo")` clears active model if it was the active one
- [ ] `delete_model("nonexistent")` returns an error
- [ ] Deleting a model with no file on disk returns a clear "not installed" message
- [ ] Deleting a custom model keeps custom metadata unless the user explicitly removes the custom entry

### GitHub registry
- [ ] GitHub registry includes the canonical Whisper.cpp entries: `tiny`, `base`, `small`, `medium`, `large-v3`, `turbo`
- [ ] Each entry has non-empty `id`, `name`, `languages`, and `url`
- [ ] Registry IDs do not need to map to `TranscriptionModel::from_id()`
- [ ] GitHub registry includes KB Whisper metadata for Swedish users
- [ ] Registry CI validates unique IDs and unique derived filenames

### Build
- [ ] `cargo check` passes
- [ ] `cargo clippy -- -D warnings` passes
- [ ] `cargo test` passes
