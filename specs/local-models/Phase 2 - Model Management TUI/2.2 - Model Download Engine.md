# 2.2 — Model Download Engine

## Problem

OSTT needs to fetch the GitHub model registry and download whisper.cpp-compatible model files from remote URLs. The download must support progress reporting, cancellation, and integrity verification. Models come from the **GitHub registry** or custom URLs.

## Objective

1. Fetch the model registry from GitHub.
2. Stream a whisper.cpp-compatible model file download from a URL with progress reporting.
3. Validate downloaded files (SHA256 when available, size check otherwise).
4. Store downloaded model files using deterministic filenames derived from model IDs.
5. Support download cancellation via callback.
6. Support custom model downloads from direct model file URLs and Hugging Face model page URLs.
7. Resolve model metadata before download so the TUI/auth flow can show name, language, size, format, URL, and compatibility.

## Specification

### Download with progress

```rust
/// Progress callback type: receives (bytes_downloaded, total_bytes, speed_mbps).
pub type DownloadProgressCallback = Box<dyn Fn(u64, u64, f64) + Send + 'static>;

/// Download a model file from a URL to a local path, streaming with progress.
pub async fn download_model(
    url: &str,
    dest_path: &Path,
    progress: Option<DownloadProgressCallback>,
) -> anyhow::Result<()> {
    let client = reqwest::Client::new();
    let response = client.get(url).send().await?;
    response.error_for_status_ref()?;

    let total_size = response.content_length().unwrap_or(0);
    let mut downloaded: u64 = 0;
    let start = std::time::Instant::now();

    let temp_path = dest_path.with_extension("tmp");
    let mut file = fs::File::create(&temp_path)?;

    let mut stream = response.bytes_stream();
    use futures_util::StreamExt;
    while let Some(chunk) = stream.next().await {
        let chunk = chunk?;
        file.write_all(&chunk)?;
        downloaded += chunk.len() as u64;

        if let Some(ref cb) = progress {
            let elapsed = start.elapsed().as_secs_f64();
            let speed_mbps = if elapsed > 0.0 {
                (downloaded as f64 / (1024.0 * 1024.0)) / elapsed
            } else {
                0.0
            };
            cb(downloaded, total_size, speed_mbps);
        }
    }

    // Rename temp → final
    fs::rename(&temp_path, dest_path)?;
    Ok(())
}
```

**Note:** `futures-util` may need to be added to `Cargo.toml` for `bytes_stream()`. Alternative: use `reqwest`'s blocking API with `std::io::copy` in a spawned thread and communicate progress via a channel.

### Post-download state

After a successful download, do not register registry models in local state. Installed status is inferred from the presence of the downloaded file in `models/files/`.

For GitHub registry models:

```rust
pub fn model_destination(entry: &RegistryEntry) -> PathBuf {
    model_files_dir().join(model_filename(&entry.id, &entry.url))
}

pub fn mark_downloaded_registry_model(entry: &RegistryEntry) -> anyhow::Result<()> {
    let path = model_destination(entry);
    if !path.exists() {
        anyhow::bail!("download completed but model file is missing at {}", path.display());
    }

    // No metadata write needed. The GitHub registry + filesystem are authoritative.
    Ok(())
}
```

For custom models that are not present in the GitHub registry, store a custom registry entry in `models.json.custom_models` so OSTT can display, re-download, and resolve it later:

```rust
pub fn register_custom_model(entry: RegistryEntry) -> anyhow::Result<()> {
    let mut state = load_state();

    state.custom_models.retain(|m| m.id != entry.id);
    state.custom_models.push(entry.clone());
    save_state(&state)
}
```

Downloading a model only installs it. Activation is handled by the calling flow. Downloading from the main model selection list can activate a model because the user is explicitly choosing it. Downloading from the local model management screen leaves the active model unchanged unless the user explicitly activates it.

### Custom model resolution

When a user pastes a direct model file URL or a Hugging Face model page URL, OSTT resolves it to a registry-shaped entry before downloading. Custom models should feel like first-class models, not opaque URLs. The resulting OSTT model ID must be filesystem-safe.

```rust
/// Resolve a direct model file URL or Hugging Face model page URL to a registry-shaped entry.
pub async fn resolve_custom_model(input: &str) -> anyhow::Result<RegistryEntry> {
    let url = Url::parse(input)
        .map_err(|_| anyhow::anyhow!("custom models require a URL"))?;

    if is_hugging_face_model_page_url(&url) {
        return resolve_hugging_face_model_page_url(url).await;
    }

    if is_direct_model_file_url(&url) {
        return resolve_direct_model_file_url(url).await;
    }

    anyhow::bail!("custom model URL must be a Hugging Face model page or a direct model file URL")
}
```

Resolution rules:
- Hugging Face model page URLs look like `https://huggingface.co/Supertone/supertonic-3`.
- Direct model file URLs point to an individual downloadable file, often under `/resolve/`.
- For Hugging Face model page URLs, query the Hugging Face model API for files.
- Prefer files that whisper.cpp can load: `.gguf`, `ggml-*.bin`, or known Whisper converted formats.
- If multiple compatible files exist, show choices in the TUI or choose the most likely default with clear confirmation.
- Extract metadata when available: model card title, tags/languages, file size, and last modified.
- Return a `RegistryEntry` with `category = "custom"`.
- Validate the downloaded file before registration by checking expected size, checksum when available, and a basic header/loadability check.
- Ask the user to confirm or edit at least `id` and `name` before saving a custom model.
- Validate custom `id` with `^[a-z0-9._-]+$`; reject `/` and other filename-unsafe characters.
- Reject custom IDs whose derived filename collides with an existing installed/registry model unless the user explicitly chooses replacement.

The custom model flow accepts URLs.

### Download cancellation

The `DownloadProgressCallback` returns a `bool` indicating whether to continue. Alternatively, use a shared `AtomicBool`:

```rust
pub struct DownloadHandle {
    pub cancel_flag: Arc<AtomicBool>,
}

impl DownloadHandle {
    pub fn cancel(&self) {
        self.cancel_flag.store(true, Ordering::SeqCst);
    }
}
```

Integration: the callback closure checks the flag and returns early if cancelled. The download loop checks after each chunk.

## Acceptance Criteria

### Registry fetch
- [ ] `fetch_registry()` fetches and parses the GitHub registry
- [ ] Registry fetch failure returns a clear network/registry error
- [ ] GitHub registry includes the canonical Whisper.cpp models

### Download
- [ ] `download_model()` streams a whisper.cpp-compatible model file to disk
- [ ] Progress callback fires at regular intervals with correct bytes/total/speed
- [ ] Download to `.tmp` is renamed to final path on success
- [ ] Corrupted/mid-way download leaves no `.tmp` file on success
- [ ] HTTP errors (404, 403) propagate as clear error messages

### Registration
- [ ] Downloading a GitHub registry model writes only the model file, not duplicate metadata in `models.json`
- [ ] Downloaded status is inferred from `models/files/{derived_filename}`
- [ ] Downloading a custom model stores a custom registry entry in `models.json.custom_models`
- [ ] Re-downloading the same model replaces the model file without duplicate state entries
- [ ] Downloading from local model management does not change the active model unless the user explicitly activates it
- [ ] Downloading from the `ostt model` selection flow may activate the downloaded model as part of the explicit selection

### Custom URLs
- [ ] `resolve_custom_model("https://huggingface.co/Supertone/supertonic-3")` treats the input as a Hugging Face model page URL
- [ ] Direct model file URLs resolve to a registry-shaped entry
- [ ] Non-URL inputs return a clear error asking for a URL
- [ ] Unknown or incompatible model page URLs return a clear error with guidance
- [ ] Resolved custom entries include at least `id`, `name`, `languages`, `size_mb`, and `url`
- [ ] KB Whisper resolves with Swedish language metadata when available
- [ ] Custom model flow asks user to confirm/edit ID and name
- [ ] Custom model IDs reject filename-unsafe characters such as `/`
- [ ] Custom model ID collisions are detected before download/registration

### Cancellation
- [ ] Calling `cancel()` on a `DownloadHandle` stops the download
- [ ] Cancelled download's `.tmp` file is cleaned up

### Build
- [ ] `cargo check` passes
- [ ] `cargo clippy -- -D warnings` passes
- [ ] `cargo test` passes
