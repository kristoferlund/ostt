# 2.3 — Model Management TUI

## Problem

Users need a visual interface for managing downloaded and available local models. This TUI is the local-model management screen inside `ostt model`: it should make the local model ecosystem understandable without requiring users to know about Whisper, GGUF, CUDA, Vulkan, or Hugging Face.

## Objective

1. Implement the local model management TUI as a screen reachable from `ostt model`.
2. The TUI has four views: **list/browse**, **download progress**, **custom model entry/resolution**, and **model info**.
3. Support keyboard navigation: activate, download, delete, view info.
4. Integrate with the model registry (spec 2.1), download engine (spec 2.2), and hardware recommendation (spec 4.1 when implemented).

## Specification

### Entry point

The local model management screen is opened from `ostt model` via the `Manage local models...` row or `[m]` shortcut. There is no separate `ostt models` command in the initial implementation.

### Data model for the TUI

```rust
#[derive(Clone)]
pub struct TuiModelEntry {
    pub id: String,
    pub name: String,
    pub description: String,
    pub size_mb: u32,
    pub is_downloaded: bool,
    pub is_active: bool,
    pub is_available_in_registry: bool,
    pub languages: Vec<String>,
    pub url: String,
    pub recommended_hardware: Option<String>,
    pub category: Option<String>,
}
```

The TUI builds its list by merging:
1. The GitHub registry entries
2. Custom registry entries stored in `models.json.custom_models`
3. The filesystem (`models/files/`) to determine installed/downloaded status, size, and timestamps

```rust
pub fn build_model_list(
    local_state: &LocalModelState,
    registry: &[RegistryEntry],
    selected_model: Option<&SelectedModel>,
) -> Vec<TuiModelEntry> {
    let mut entries: Vec<TuiModelEntry> = Vec::new();
    let all_entries = registry.iter().chain(local_state.custom_models.iter());

    for reg in all_entries {
        let path = model_files_dir().join(model_filename(&reg.id, &reg.url));
        let is_downloaded = path.exists();
        let is_active = selected_model
            .map(|selected| selected.provider_id == "local" && selected.model_id == reg.id)
            .unwrap_or(false);
        entries.push(TuiModelEntry {
            id: reg.id.to_string(),
            name: reg.name.to_string(),
            description: reg.description.to_string(),
            size_mb: reg.size_mb,
            is_downloaded,
            is_active,
            is_available_in_registry: registry.iter().any(|entry| entry.id == reg.id),
            languages: reg.languages.clone(),
            url: reg.url.clone(),
            recommended_hardware: reg.recommended_hardware.clone(),
            category: reg.category.clone(),
        });
    }

    entries
}
```

### App state and rendering

```rust
pub struct ModelTui {
    /// All model entries (merged from local + registry)
    entries: Vec<TuiModelEntry>,
    /// Currently selected index
    selected: usize,
    /// Current view mode
    mode: TuiMode,
    /// Download state (if downloading)
    download: Option<DownloadState>,
    /// Terminal rendering state
    terminal: Terminal<CrosstermBackend<Stdout>>,
}

enum TuiMode {
    /// Main list view
    Browse,
    /// Custom model URL input
    CustomModelInput { input: String },
    /// Download in progress
    Downloading { model_id: String, progress: f64, speed: f64, status: String },
    /// Model detail view
    Info { entry: TuiModelEntry },
}
```

### Keyboard bindings

| Key | Action |
|-----|--------|
| `↑` / `↓` | Navigate the list |
| `Enter` | Activate the selected model (if downloaded), or show error (if not) |
| `d` | Start download of the selected model (if available in registry) |
| `r` | Remove the selected model (if downloaded), with confirmation prompt |
| `i` | Show info panel for the selected model |
| `c` | Switch to custom model URL input |
| `Esc` | Go back / cancel input / close info panel |
| `q` | Quit the TUI |

### Custom model input view

```
┌────────── Download Custom Model ──────────────────┐
│                                                    │
│  Enter model page or file URL:                     │
│  ┌──────────────────────────────────────────────┐  │
│  │ https://huggingface.co/Supertone/supertonic-3│  │
│  └──────────────────────────────────────────────┘  │
│                                                    │
│  Info: KB Whisper Large (Swedish)                  │
│  Size: ~1.6 GB                                     │
│                                                    │
│  [Enter] Confirm  [Esc] Cancel                     │
└────────────────────────────────────────────────────┘
```

Implementation: a simple text input using `tui-input` crate (already a dependency). When the user presses Enter, call `resolve_custom_model()` (spec 2.2), show a resolution/confirmation panel with model info, then start download.

The custom model flow must support both:
- Hugging Face model page URLs, e.g. `https://huggingface.co/Supertone/supertonic-3`
- Direct model file URLs, e.g. a `huggingface.co/.../resolve/...` URL

It should not require KB Whisper or other regional models to be hardcoded in the binary. The registry can provide good defaults; the resolver should handle compatible unknown models.

### Interaction flow

```
Open TUI
  → Build model list from local state + registry
  → Render list view
  ↓
  [d] on entry → start download → download view → complete → refresh list
  [Enter] on downloaded → activate → refresh list (active indicator moves)
  [r] → confirm → delete → refresh list
  [i] → info view → [Esc] back to list
  [c] → custom input → enter URL → resolve → confirm → download view → complete
  [q] → quit
```

### Offline handling

- If the GitHub registry is unreachable, the TUI shows a clear registry/network error
- The user can still add a custom model by URL/repo ID if that flow does not depend on the registry response
- Downloaded custom models are visible from `models.json.custom_models`

## Acceptance Criteria

### Navigation
- [ ] TUI opens from the local model management entry in `ostt model`
- [ ] TUI can be reused or embedded from the `Local` branch of `ostt model`
- [ ] Arrow keys navigate the model list
- [ ] Enter activates a downloaded model
- [ ] Enter on a non-downloaded model shows a clear message ("Download first with [d]")
- [ ] `q` quits the TUI cleanly (terminal restored)
- [ ] `Esc` works from all sub-views to go back

### Download
- [ ] `[d]` on an available model starts download with progress gauge
- [ ] Progress gauge updates with bytes/speed/ETA
- [ ] Tab cancels an in-progress download
- [ ] On completion, the model appears in the "downloaded" section

### Deletion
- [ ] `[r]` shows a confirmation dialog
- [ ] Confirming deletes the model file and removes from state
- [ ] Cancelling confirmation returns to the list
- [ ] Deleting an active model automatically deactivates it

### Custom models
- [ ] `[c]` opens the URL input view
- [ ] Enter resolves a Hugging Face model page URL or direct file URL and shows confirmation metadata before download
- [ ] Invalid URLs show an error message
- [ ] Hugging Face model page URLs resolve to first-class custom models with safe OSTT IDs

### Info
- [ ] `[i]` shows model details including size, download date, path
- [ ] Info includes download URL, languages, and recommendation metadata when available
- [ ] [Esc] closes the info view

### State synchronization
- [ ] Activation persists through OSTT's selected-model mechanism
- [ ] Custom model entries persist to `models.json.custom_models`
- [ ] GitHub registry model downloads do not duplicate metadata into `models.json`
- [ ] TUI correctly reflects the active model (marked with ◉)
- [ ] TUI shows disk usage summary from filesystem metadata
