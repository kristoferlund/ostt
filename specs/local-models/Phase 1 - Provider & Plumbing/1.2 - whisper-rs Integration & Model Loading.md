# 1.2 — whisper-rs Integration & Model Loading

## Problem

The `local` provider module is a stub (spec 1.1). It needs real whisper-rs integration: loading a whisper.cpp-compatible model from disk, validating that the recorded audio is already in the required local format, running inference, filtering obvious hallucinations, and returning transcribed text. The module must handle model file lookup, audio format validation, sample loading, and the whisper-rs API lifecycle.

## Objective

1. Implement `local::transcribe()` to load a whisper.cpp-compatible model and transcribe audio.
2. Model path resolution: look up the selected local model ID from OSTT's selected-model state and find its installed file on disk.
3. Audio format validation: local transcription requires OSTT recordings to be saved as mono 16 kHz WAV/PCM via the existing audio output settings.
4. Error handling: missing model file, corrupt model, empty transcription, whisper-rs errors.
5. Anti-hallucination filtering for blank/silent/garbled output, matching the Vocalinux parity requirement.

## Specification

### Model path resolution

Add a helper module `src/transcription/local_models.rs` (shared between the API module and future model management code):

```rust
/// Returns the directory where models are stored.
pub fn models_dir() -> PathBuf {
    let base = dirs::data_dir().unwrap_or_else(|| PathBuf::from("~/.local/share"));
    base.join("ostt").join("models")
}

/// Error type for model-related failures.
#[derive(Debug, thiserror::Error)]
pub enum ModelError {
    #[error("Model '{0}' is not downloaded. Run `ostt models download {0}` first.")]
    NotDownloaded(String),
    #[error("Model file not found at {0}")]
    FileNotFound(PathBuf),
    #[error("Failed to load model: {0}")]
    LoadFailed(String),
}
```

Model path resolution uses the GitHub registry, custom model entries, and the local model file directory. Installed registry models are discovered by deriving the expected filename from the model ID and registry entry, then checking `~/.local/share/ostt/models/files/`. Custom model entries are stored in `models.json`.

Minimum state shape needed by this spec:

```rust
#[derive(Debug, Clone, Deserialize)]
pub struct LocalModelState {
    pub custom_models: Vec<RegistryEntry>,
}
```

Active provider/model selection uses OSTT's selected-model state in `~/.local/share/ostt/model`. Local support extends that mechanism so selecting `provider = local` + model ID follows the same path as selecting Deepgram/Nova or OpenAI/Whisper. Because local model IDs are data-driven and may not exist in `TranscriptionModel`, the selected-model state stores both provider ID and model ID.

Phase 1 implements registry loading, custom model entry loading, `model_filename(id, url)`, and installed model path resolution. Phase 2 adds custom model entry writes, deletion, and persistence helpers.

Lookup logic in `local::transcribe()`:

1. Read the selected local model ID from OSTT's existing selected-model state (e.g. `"turbo"` or `"kb-whisper-large"`)
2. Find the model entry in the GitHub registry or `models.json.custom_models`
3. Derive the expected file path: `models/files/{model_filename(entry.id, entry.url)}`
4. Check if the file exists → `ModelError::NotDownloaded` if not
5. Pass the path to whisper-rs

Model metadata comes from registry/custom entries. Installed status, size, and timestamps come from the filesystem.

### Audio format requirements

OSTT records through the existing configurable ffmpeg-based output pipeline:

- `audio.sample_rate` controls capture sample rate.
- `audio.output_format` controls codec and ffmpeg options.
- `recording/audio.rs` forces mono with `-ac 1` during conversion.

The local provider should expect OSTT recordings to be in the format whisper.cpp can consume efficiently:

- **Container:** WAV
- **Codec/sample format:** signed 16-bit PCM (`pcm_s16le`)
- **Sample rate:** 16,000 Hz
- **Channels:** mono

Recommended OSTT audio config for local transcription:

```toml
[audio]
sample_rate = 16000
output_format = "pcm_s16le -ar 16000"
```

`pcm_s16le` maps to `.wav` in the current record flow.

The local provider should validate the input file before transcription. If the selected model is local and the recording format is incompatible, return a clear error that explains the required `audio.output_format` and suggests updating config. The `ostt model` wizard may also offer to update the audio settings when the user selects a local model.

After validation, load samples from the WAV file into the `Vec<f32>` format expected by whisper-rs. This is decoding/reading, not format conversion or resampling.

Open product decision: change OSTT's default `audio.output_format` from compressed MP3 to local-friendly WAV/PCM, since local models may become the common path. That decision affects cloud users too because recordings become larger, so it should be made deliberately and reflected in config defaults.

For `ostt transcribe <FILE>`, where the input may be arbitrary user audio, incompatible files return guidance. A future explicit preflight/conversion path can support arbitrary input files.

### whisper-rs inference

In `transcription/api/local.rs`:

```rust
use whisper_rs::WhisperContext;

pub async fn transcribe(config: &TranscriptionConfig, audio_path: &Path) -> anyhow::Result<String> {
    let model_id = config.model.id();
    let model_path = resolve_model_path(model_id)?;

    validate_local_audio_format(audio_path)?;

    // Load audio samples from already-compatible WAV/PCM.
    let audio_samples = load_audio_for_whisper(audio_path)?;

    // Load model (this is CPU-heavy — consider offloading to a blocking task)
    let ctx = tokio::task::spawn_blocking(move || {
        let ctx = WhisperContext::new(&model_path.to_string_lossy())
            .map_err(|e| anyhow::anyhow!("Failed to load whisper model: {}", e))?;

        // Run transcription
        let mut state = ctx.create_state()
            .map_err(|e| anyhow::anyhow!("Failed to create whisper state: {}", e))?;

        state.full(&ctx, &audio_samples, |_| {})
            .map_err(|e| anyhow::anyhow!("Transcription failed: {}", e))?;

        // Collect segments
        let num_segments = state.full_n_segments();
        let mut text = String::new();
        for i in 0..num_segments {
            let segment = state.full_get_segment_text(i)
                .map_err(|e| anyhow::anyhow!("Failed to get segment {}: {}", i, e))?;
            text.push_str(&segment);
            text.push(' ');
        }

        Ok::<String, anyhow::Error>(text.trim().to_string())
    }).await??;

    Ok(text)
}
```

Key considerations:
- `WhisperContext::new()` is blocking (file I/O + model loading). Must run in `spawn_blocking`.
- Model loading takes 1-3 seconds depending on model size.
- The `full()` method transcribes the entire audio buffer at once (batch mode). Streaming can come later.
- The `|_| {}` closure is for segment-level callbacks — unused in batch mode.
- Audio samples must be `Vec<f32>` normalized to ~[-1.0, 1.0] range.
- Backend acceleration depends on how whisper.cpp/whisper-rs is built. This spec only requires CPU correctness; spec 3.3 must define distribution/build strategy for CUDA/Vulkan/Metal before promising GPU acceleration.

### Anti-hallucination filtering

Local Whisper models may emit junk for silence, background noise, or very short clips. Vocalinux handles this well; OSTT should include the same baseline behavior from the first usable local provider.

```rust
fn filter_obvious_hallucination(text: &str) -> Option<String> {
    let trimmed = text.trim();
    if trimmed.is_empty() {
        return None;
    }

    let lower = trimmed.to_lowercase();
    let blocked = ["[blank_audio]", "[silence]", "♪", "♫"];
    if blocked.iter().any(|token| lower.contains(token)) {
        return None;
    }

    let alphanumeric = trimmed.chars().filter(|c| c.is_alphanumeric()).count();
    let total = trimmed.chars().filter(|c| !c.is_whitespace()).count().max(1);
    if (alphanumeric as f32 / total as f32) < 0.30 {
        return None;
    }

    Some(trimmed.to_string())
}
```

Return an empty string for filtered output so behavior matches silent/empty cloud results.

### Parameter passthrough

The whisper-rs API supports various inference parameters. For MVP, use sensible defaults defined in config:

```rust
pub struct WhisperParams {
    pub no_timestamps: bool,       // default: true (we don't need timestamps)
    pub no_context: bool,          // default: true
    pub temperature: f32,          // default: 0.0 (greedy sampling)
    pub entropy_thold: f32,        // default: 2.4
    pub no_speech_thold: f32,      // default: 0.6
}
```

These are populated from config (spec 3.1) but for now hardcoded to defaults.

### Error handling

| Scenario | Behavior |
|----------|----------|
| Model file not found | Return `ModelError::NotDownloaded` with clear message + instruction to run `ostt models download` |
| whisper-rs load fails (corrupt/old model) | Return `ModelError::LoadFailed` with the whisper-rs error message |
| Incompatible audio format | Return clear error with required local audio config |
| Empty result (silent audio) | Return empty string (same as cloud providers) |
| whisper-rs returns obvious hallucination/garbage | Filter and return empty string |

## Acceptance Criteria

### Path resolution
- [ ] Model path resolves from registry/custom metadata plus derived filename
- [ ] Canonical model ID `turbo` resolves when installed
- [ ] Custom/regional model ID `kb-whisper-large` resolves when installed
- [ ] Missing model returns `ModelError::NotDownloaded` with actionable message

### Audio format
- [ ] Local provider validates input is WAV/PCM 16 kHz mono-compatible before transcription
- [ ] Incompatible audio returns a clear error with suggested `audio.output_format = "pcm_s16le -ar 16000"`
- [ ] `ostt model` local branch can warn/offer config update if current audio output is MP3 or otherwise incompatible
- [ ] Local transcription uses OSTT's existing audio output settings for local-compatible recordings

### Transcription
- [ ] Given a valid model file and audio, returns transcribed text
- [ ] Empty/silent audio returns empty string
- [ ] Known blank/silence tokens and low-alphanumeric junk are filtered to empty string
- [ ] Invalid model file returns a load error

### Build
- [ ] `cargo build` compiles with `whisper-rs` in the standard dependency set (requires cmake/C++ compiler for source builds)
- [ ] `cargo clippy -- -D warnings` passes
- [ ] `cargo test` passes
