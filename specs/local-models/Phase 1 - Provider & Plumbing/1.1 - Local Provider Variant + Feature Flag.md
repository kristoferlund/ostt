# 1.1 — Local Provider Variant + Standard Build

## Problem

OSTT currently supports only cloud transcription providers (OpenAI, Deepgram, etc.). There is no `Local` variant in the `TranscriptionProvider` enum and no way to select on-device transcription. Local transcription is part of the standard OSTT build.

Adding local inference requires:
1. A new provider enum variant
2. A standard `whisper-rs` dependency bundled into normal release builds
3. A data-driven local model identifier, because local models are not a closed enum: the user may select canonical Whisper models, regional fine-tunes such as KB Whisper, or arbitrary compatible Hugging Face models.
4. A placeholder module for the provider implementation

## Objective

1. Add `whisper-rs` to the standard dependency set.
2. Add `Local` variant to `TranscriptionProvider` enum.
3. Do **not** add one `TranscriptionModel` enum variant per local model. Keep local model identity as a string stored in config/model state.
4. Create the `transcription/api/local.rs` module with a stub implementation.
5. Wire the new variant into the dispatch in `transcription/api/mod.rs`.

## Specification

### Standard dependency in `Cargo.toml`

```toml
[dependencies]
whisper-rs = "0.14"
```

The local provider is compiled into normal OSTT builds.

`whisper-rs` (v0.14) requires cmake and a C++ compiler at build time for source builds. End users installing pre-built binaries do not need those build tools. AUR/source-build users will need the normal native build dependencies documented by the package.

Backend acceleration features such as CUDA/Vulkan/Metal are separate packaging questions covered in spec 3.3. CPU local transcription is the standard baseline.

### New `TranscriptionProvider` variant

In `src/transcription/provider.rs`:

```rust
pub enum TranscriptionProvider {
    // existing variants...
    Local,
}
```

Add to `id()`: `"local"`, `name()`: `"Local (whisper.cpp)"`, `from_id()`: `"local" => Some(Self::Local)`.

Add `Local` to the `all()` list.

### Local model identity

Do **not** extend `TranscriptionModel` with static local model variants such as `LocalTurbo` or `LocalLargeV3`. That would conflict with the research direction: OSTT must support a growing ecosystem of registry and custom Hugging Face models without a new binary release for every model.

Instead, local model IDs are strings stored in config and local model state:

```toml
[transcription]
provider = "local"
model = "turbo"                  # canonical registry ID

[transcription.local]
language = "auto"
```

Valid local model IDs are filesystem-safe strings matching `^[a-z0-9._-]+$`. Examples:
- Canonical registry IDs: `tiny`, `base`, `small`, `medium`, `large-v3`, `turbo`
- Registry IDs for regional/community models: `kb-whisper-large`, `nb-whisper-large`, etc.
- User-added custom Hugging Face models once resolved and assigned a safe OSTT model ID

Hugging Face repo IDs such as `KBLab/kb-whisper-large` are resolved to safe OSTT model IDs such as `kb-whisper-large`.

If the existing code requires `TranscriptionConfig.model` to parse into `TranscriptionModel`, introduce a local-specific model string escape hatch rather than encoding every local model in the enum. The important invariant is: **cloud models can stay enum-backed; local models are registry-backed.**

### Module: `src/transcription/api/local.rs`

Create a new file with the same signature pattern as existing providers:

```rust
use std::path::Path;
use super::TranscriptionConfig;

pub async fn transcribe(config: &TranscriptionConfig, audio_path: &Path) -> anyhow::Result<String> {
    anyhow::bail!("local provider not yet implemented — this is a stub")
}
```

The full implementation comes in spec 1.2.

### Dispatch wiring in `src/transcription/api/mod.rs`

Add `mod local;` and a match arm:

```rust
TranscriptionProvider::Local => local::transcribe(config, audio_path).await,
```

The local module is declared like any other provider module. Runtime errors should be about missing models, incompatible model files, or unavailable accelerated backends.

## Acceptance Criteria

### Standard build
- [ ] `cargo build` succeeds with `whisper-rs` compiled into the normal binary
- [ ] Source builds document required native build dependencies (`cmake`, C++ compiler, etc.)

### Provider enum
- [ ] `TranscriptionProvider::Local` exists
- [ ] `provider.id()` returns `"local"`
- [ ] `provider.name()` returns `"Local (whisper.cpp)"`
- [ ] `TranscriptionProvider::from_id("local")` returns `Some(Local)`
- [ ] `TranscriptionProvider::all()` includes `Local`

### Local model identity
- [ ] No static `TranscriptionModel` variants are added for individual local models
- [ ] `provider = "local"` can carry arbitrary local model IDs from config/state
- [ ] Canonical IDs (`tiny`, `base`, `small`, `medium`, `large-v3`, `turbo`) are accepted through the local model registry
- [ ] Custom registry/Hugging Face IDs are not blocked by enum parsing

### Dispatch
- [ ] Selecting `provider = "local"` routes to `local::transcribe` in the standard build
- [ ] Local provider errors are runtime/model errors

### Build
- [ ] `cargo check` passes
- [ ] `cargo clippy -- -D warnings` passes
- [ ] `cargo test` passes

## Files Modified

- `Cargo.toml` — standard `whisper-rs` dependency
- `src/transcription/provider.rs` — new variant
- `src/transcription/model.rs` or config conversion code — only if needed to avoid enum-blocking local model IDs
- `src/transcription/api/mod.rs` — module declaration + dispatch arm
- `src/transcription/api/local.rs` — new file (stub)
