# 3.1 — Local Provider Config Types

## Problem

The `ostt.toml` config file currently supports `[audio]` and `[providers]` sections but has no way to configure local transcription behavior. Active provider/model selection is stored outside `ostt.toml` via OSTT's selected-model mechanism, so this config contains local provider behavior: language hints, whisper.cpp inference parameters, daemon behavior, optionally model storage path, and per-model overrides.

## Objective

1. Add local provider settings to the existing config schema.
2. Define Rust types for local-specific config with serde deserialization.
3. Wire the config into the existing `OsttConfig` struct.
4. Provide defaults so that selecting a local model works with zero additional config.
5. Support per-model local settings overrides.

## Specification

### TOML schema

```toml
# ~/.config/ostt/ostt.toml

[providers.local]
# Language hint ("auto" for auto-detect, or ISO code like "sv", "en", "fr")
language = "auto"

# whisper.cpp inference parameters (optional, defaults shown)
no_timestamps = true
no_context = true
temperature = 0.0
entropy_thold = 2.4
no_speech_thold = 0.6

# Custom model storage path (optional)
# models_path = "/media/ssd/ostt-models"

# Keep the active model warm for faster repeated transcriptions
daemon = true
daemon_idle_timeout_secs = 300

# Optional per-model overrides. Quote model IDs to preserve exact keys.
[providers.local.models."kb-whisper-large"]
language = "sv"
no_context = false

[providers.local.models."turbo"]
language = "auto"
```

### Rust types

In `src/config/file.rs`:

```rust
use std::collections::HashMap;

/// Local transcription provider configuration.
#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(default)]
pub struct LocalTranscriptionConfig {
    /// Language hint: "auto" or ISO code (e.g. "sv", "en")
    pub language: String,
    /// Suppress timestamp output
    pub no_timestamps: bool,
    /// Suppress text context from previous segments
    pub no_context: bool,
    /// Sampling temperature (0.0 = greedy/ deterministic)
    pub temperature: f32,
    /// Entropy threshold for fallback (2.4 = default)
    pub entropy_thold: f32,
    /// No-speech probability threshold (0.6 = default)
    pub no_speech_thold: f32,
    /// Optional override for model storage directory
    pub models_path: Option<String>,
    /// Auto-start a local daemon and keep the model warm between transcriptions
    pub daemon: bool,
    /// Idle timeout before the daemon unloads the model and exits
    pub daemon_idle_timeout_secs: u64,
    /// Per-model overrides keyed by local model ID.
    #[serde(default)]
    pub models: HashMap<String, LocalModelOverride>,
}

/// Optional overrides for one local model.
#[derive(Debug, Clone, Default, Serialize, Deserialize)]
pub struct LocalModelOverride {
    pub language: Option<String>,
    pub no_timestamps: Option<bool>,
    pub no_context: Option<bool>,
    pub temperature: Option<f32>,
    pub entropy_thold: Option<f32>,
    pub no_speech_thold: Option<f32>,
    pub daemon: Option<bool>,
    pub daemon_idle_timeout_secs: Option<u64>,
}

impl Default for LocalTranscriptionConfig {
    fn default() -> Self {
        Self {
            language: "auto".to_string(),
            no_timestamps: true,
            no_context: true,
            temperature: 0.0,
            entropy_thold: 2.4,
            no_speech_thold: 0.6,
            models_path: None,
            daemon: true,
            daemon_idle_timeout_secs: 300,
            models: HashMap::new(),
        }
    }
}
```

At transcription time, resolve effective settings by merging global local config with the selected model override:

```rust
impl LocalTranscriptionConfig {
    pub fn effective_for_model(&self, model_id: &str) -> EffectiveLocalConfig {
        let override_config = self.models.get(model_id);
        EffectiveLocalConfig {
            language: override_config.and_then(|m| m.language.clone()).unwrap_or_else(|| self.language.clone()),
            no_timestamps: override_config.and_then(|m| m.no_timestamps).unwrap_or(self.no_timestamps),
            no_context: override_config.and_then(|m| m.no_context).unwrap_or(self.no_context),
            temperature: override_config.and_then(|m| m.temperature).unwrap_or(self.temperature),
            entropy_thold: override_config.and_then(|m| m.entropy_thold).unwrap_or(self.entropy_thold),
            no_speech_thold: override_config.and_then(|m| m.no_speech_thold).unwrap_or(self.no_speech_thold),
            daemon: override_config.and_then(|m| m.daemon).unwrap_or(self.daemon),
            daemon_idle_timeout_secs: override_config
                .and_then(|m| m.daemon_idle_timeout_secs)
                .unwrap_or(self.daemon_idle_timeout_secs),
        }
    }
}
```

### Integration with existing config

Extend the existing `ProvidersConfig` with a local provider section:

```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct ProvidersConfig {
    // existing provider config fields...
    #[serde(default)]
    pub local: LocalTranscriptionConfig,
}
```

The `local` section is always present in the struct (with defaults) but only affects local transcription.

Active provider/model selection remains in OSTT's selected-model state (`~/.local/share/ostt/model` today), not in `ostt.toml` and not in `models.json`. That selected-model state should be extended to include provider ID because local model IDs are data-driven and cannot always be mapped through `TranscriptionModel::from_id()`.

### Validation

Validate local config values at config load:

```rust
impl LocalTranscriptionConfig {
    pub fn validate(&self) -> anyhow::Result<()> {
        if self.temperature < 0.0 || self.temperature > 1.0 {
            anyhow::bail!("temperature must be between 0.0 and 1.0");
        }
        if self.entropy_thold < 0.0 {
            anyhow::bail!("entropy_thold must be >= 0.0");
        }
        if self.no_speech_thold < 0.0 || self.no_speech_thold > 1.0 {
            anyhow::bail!("no_speech_thold must be between 0.0 and 1.0");
        }
        if let Some(ref path) = self.models_path {
            let p = Path::new(path);
            if !p.exists() {
                anyhow::bail!("models_path '{}' does not exist", path);
            }
        }
        if self.daemon_idle_timeout_secs < 30 {
            anyhow::bail!("daemon_idle_timeout_secs must be at least 30 seconds");
        }
        for (model_id, override_config) in &self.models {
            validate_model_id(model_id)?;
            override_config.validate()?;
        }
        Ok(())
    }
}
```

Model ID validation is not part of config validation because active model selection is not in `ostt.toml`. Local model IDs are checked against registry/custom state at use time.

Call validation from the config loading path.

### Re-exports

```rust
// src/config/mod.rs
pub use file::LocalTranscriptionConfig;
```

### Using the config

The transcription flow should pass both selected provider/model and local config into the local provider. Add a helper to access local params:

```rust
impl super::TranscriptionConfig {
    /// Returns the local-specific config if the provider is local.
    pub fn local_config(&self) -> Option<&LocalTranscriptionConfig> {
        if self.provider == TranscriptionProvider::Local {
            Some(&self.providers.local)
        } else {
            None
        }
    }
}
```

Alternatively, just pass the `LocalTranscriptionConfig` directly into `local::transcribe()`.

## Acceptance Criteria

### Config parsing
- [ ] A config with `[providers.local]` section deserializes correctly
- [ ] A config without `[providers.local]` defaults to standard local params
- [ ] All whisper.cpp params deserialize from TOML
- [ ] `models_path` is optional and defaults to `None`
- [ ] `daemon` defaults to `true`
- [ ] `daemon_idle_timeout_secs` defaults to `300`
- [ ] `language` defaults to `"auto"`
- [ ] Per-model overrides deserialize from `[providers.local.models."<model-id>"]`
- [ ] Effective config merges global defaults with selected model override

### Validation
- [ ] temperature > 1.0 fails validation
- [ ] temperature < 0.0 fails validation
- [ ] entropy_thold < 0.0 fails validation
- [ ] no_speech_thold > 1.0 fails validation
- [ ] Valid values pass validation
- [ ] Non-existent `models_path` fails validation at config load
- [ ] Config validation does not validate active model ID
- [ ] Per-model override keys are validated as safe local model IDs
- [ ] Per-model override values use the same validation rules as global local config

### Integration
- [ ] Local transcription can run with full defaults even with no `[providers.local]` section
- [ ] Non-local providers ignore the `local` config entirely
- [ ] `local_config()` returns `Some` when provider is local
- [ ] Active provider/model selection remains in OSTT's selected-model mechanism
- [ ] Re-exports compile from `src/config/mod.rs`

### Build
- [ ] `cargo check` passes
- [ ] `cargo clippy -- -D warnings` passes
- [ ] `cargo test` passes
