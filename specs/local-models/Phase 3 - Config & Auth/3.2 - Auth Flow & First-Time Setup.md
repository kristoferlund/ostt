# 3.2 — Auth Flow & First-Time Setup

## Problem

OSTT currently treats authentication and model selection as one combined setup flow, implemented outside the richer Ratatui UI system. That works while the model list is short, but it does not scale once local models, registry models, downloaded models, and many cloud provider models are all available. The primary user intent is not "authenticate"; it is "choose the model OSTT should use." Authentication is a prerequisite for some model choices.

The setup flow should more closely mimic OpenCode-style ergonomics: `ostt model` is the canonical interactive Ratatui interface for choosing the active model. Local model management is reachable from that interface. Authentication remains a small Cliclack flow.

There are three relevant states to handle:

1. No provider credentials and no model selected.
2. Some provider credentials exist, but no active model is selected.
3. A selected model can no longer be used because credentials or local files were removed after selection.

OSTT should not ship with a preinstalled or preselected model. Many users will want online providers, and local models are too large to bundle casually. First run should produce a clear, actionable message instead of guessing.

## Objective

1. Make `ostt model` the canonical interactive Ratatui model-selection flow.
2. Show authenticated cloud provider models and local models in one selectable list.
3. Provide a path from `ostt model` into local model management.
4. Keep `ostt auth login` and `ostt auth logout` as simple Cliclack credential commands.
5. On first run without configuration, show a clear error that points to `ostt model` and `ostt auth login`.
6. Save active provider/model through OSTT's selected-model mechanism.

## Specification

### Command shape

Preferred command structure:

```bash
ostt model               # Ratatui model selection + local model management entry point

ostt auth login          # Cliclack credential setup
ostt auth logout         # Cliclack credential removal
```

Product distinction:
- **`ostt model`** chooses what OSTT will use now and provides access to local model management.
- **`ostt auth login/logout`** manages cloud provider credentials.

No `ostt local` command and no separate `ostt models` command are part of the initial implementation.

### First-run and Recovery Behavior

OSTT should not silently choose a provider/model. If the user runs `ostt` or `ostt record` without an active model, return an actionable error:

```text
No transcription model selected.

Run `ostt model` to choose an online or local transcription model.
Run `ostt auth login` first to add credentials for cloud providers.
```

This state normally happens only if the user authenticated, selected a cloud model, and later logged out of that provider:

```text
OpenAI is selected, but no OpenAI API key is configured.

Run `ostt auth login` and choose OpenAI.
```

This state normally happens only if the model file was manually deleted after selection:

```text
Local model `turbo` is selected but not downloaded.

Run `ostt model` to download or select a local model.
```

### `ostt model` Ratatui flow

`ostt model` opens a Ratatui interface that follows the same visual language as existing OSTT TUI screens such as history.

Main screen: choose active model.

```text
Choose transcription model

OpenAI
  ○ gpt-4o-transcribe
  ○ whisper-1

Groq
  ○ whisper-large-v3-turbo

Local
  ◉ turbo (809 MB) — Best speed/accuracy ratio  [recommended]
  ○ KB Whisper Large (1.6 GB) — Swedish dictation
  ○ small (466 MB) — Balanced speed/accuracy

  Manage local models...

[↑↓] Navigate  [Enter] Select  [m] Manage local  [q] Quit
```

Rules:
- Cloud provider sections show only authenticated providers.
- Local shows downloaded models plus registry models available for download.
- Selecting a downloaded model saves active provider/model through OSTT's selected-model mechanism.
- Selecting a local model that is not downloaded starts a download confirmation/progress flow; after download, the user can activate it.
- `Manage local models...` opens the local model management screen described in spec 2.3. Spec 2.3 owns local management details.
- The local flow checks current audio settings and can offer to update `audio.output_format` to `pcm_s16le -ar 16000`.

Required navigation behavior:
- `Esc` goes back one step when possible.
- `q` quits the wizard without modifying config unless a prior explicit save already happened.
- Local management screens have a visible back path to the main model list.

### Auth Login Flow

`ostt auth login` is a Cliclack flow:

1. Select cloud provider.
2. Enter API key.
3. Store credential.
4. Show completion message with guidance to run `ostt model`.

Local is not shown in `auth login` because it has no credential.

### Auth Logout Flow

`ostt auth logout` removes credentials for one selected cloud provider:

1. Show providers that currently have stored credentials.
2. User selects a provider.
3. Confirm removal.
4. Delete only that provider's credential.
5. If the active model uses that provider, clear the active model and show guidance to run `ostt model`.

Local is not shown in `auth logout` because it has no credential.

### Model Selection UI

The model selection flow shows models grouped by provider:

```text
Choose transcription model:

OpenAI
  ○ gpt-4o-transcribe
  ○ whisper-1

Groq
  ○ whisper-large-v3-turbo

Local
  ◉ turbo (809 MB) — Best speed/accuracy ratio  [recommended]
  ○ KB Whisper Large (1.6 GB) — Swedish dictation
  ○ small (466 MB) — Balanced speed/accuracy
  ○ Custom Hugging Face model...
```

Cloud provider sections appear only when the user has credentials for that provider. Local models appear from the GitHub registry and custom local state.

The local management screen is specified in detail in spec 2.3. At a high level, it supports downloaded/available grouping, registry downloads, custom URL entry, cancellation, deletion, and model info.

### Download progress

Local model downloads show progress inside the Ratatui flow:

```
Downloading Whisper Turbo (809 MB)...
  ████████████░░░░░░░░  45%  •  12.4 MB/s  •  ETA: 36s
```

### Config write

After completing the model selection flow, save the active provider/model through OSTT's selected-model mechanism. Today OSTT stores only the model ID in `~/.local/share/ostt/model` and infers provider from `TranscriptionModel`; local support should extend this to preserve both provider ID and model ID so arbitrary local model IDs work.

Conceptual selected-model state:

```toml
provider = "local"
model = "turbo"
```

Local behavior settings such as language and daemon configuration belong in config, but active provider/model selection should reuse the existing selected-model path rather than `models.json`.

## Acceptance Criteria

- [ ] Running `ostt`/`ostt record` with no active model returns a clear actionable setup error pointing to `ostt model`
- [ ] `ostt model` opens a Ratatui wizard matching existing OSTT TUI visual patterns
- [ ] `ostt model` shows models grouped under authenticated cloud providers and Local
- [ ] `ostt model` provides a visible entry point to local model management
- [ ] Selecting a downloaded model saves active provider/model
- [ ] Selecting a missing local model can download it, then activate it only after explicit selection
- [ ] Local management can browse GitHub registry models, download models, cancel downloads, remove downloaded models, inspect model details, and add custom model URLs
- [ ] Local flow warns/offers config update when current audio output is incompatible with local transcription
- [ ] `ostt auth login` lets the user select a provider
- [ ] `ostt auth logout` removes credentials for one selected provider
- [ ] Logout clears active model if it depended on the removed provider
- [ ] `ostt auth login/logout` use Cliclack, not Ratatui
- [ ] Local is not listed in auth login/logout
- [ ] User can enter a custom Hugging Face model page URL or direct model file URL
- [ ] KB Whisper appears when present in the GitHub registry
- [ ] Download can be cancelled with Ctrl+C
- [ ] On local model selection, selected-model state is written with `provider = "local"` and the local model ID
- [ ] On cloud model selection, selected-model state is written with the selected provider/model
- [ ] Config writes the selected local model ID as a string, not a `TranscriptionModel` enum variant
- [ ] `ostt auth login` preserves existing config fields for other providers
