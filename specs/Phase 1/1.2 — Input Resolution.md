# 1.2 тАФ Input Resolution

## Problem

AI action inputs can reference dynamic sources (`"transcription"`, `"keywords"`) or external files (`file` field). These need to be resolved at runtime into concrete message strings before being sent to an LLM API.

## Objective

1. Create a function that takes a list of `ActionInput` entries and resolves them into a list of `(role, content)` string pairs.
2. Handle all three content sources: literal `content`, `source` references, and `file` paths.

## Specification

### New module: `src/process/input.rs`

Also create `src/process/mod.rs` as the parent module.

```rust
/// A resolved message ready to send to an LLM.
pub struct ResolvedMessage {
    pub role: String,
    pub content: String,
}

/// Resolves action inputs into concrete messages.
///
/// # Arguments
/// * `inputs` - The action's input configuration
/// * `transcription` - The transcribed text to substitute for `source = "transcription"`
/// * `keywords` - The user's keyword list to substitute for `source = "keywords"`
///
/// # Errors
/// - If a `file` path cannot be read
/// - If an input has no content source (should be caught by validation, but defensive)
pub fn resolve_inputs(
    inputs: &[ActionInput],
    transcription: &str,
    keywords: &[String],
) -> anyhow::Result<Vec<ResolvedMessage>>
```

### Resolution logic

For each `ActionInput`:

1. If `content` is `Some(text)` тАФ use `text` as-is.
2. If `source` is `Some("transcription")` тАФ use the `transcription` argument.
3. If `source` is `Some("keywords")` тАФ join keywords with newlines. If keywords list is empty, skip this input entirely (don't send an empty message).
4. If `file` is `Some(path)` тАФ read the file at `path`. Expand `~` to the user's home directory. Return error if file doesn't exist or can't be read.

If an input somehow has no resolvable content (all fields None), skip it with a warning log.

### Module registration

Add `pub mod process;` to `src/lib.rs`.

## Acceptance Criteria

- [ ] Literal `content` inputs resolve to the exact string
- [ ] `source = "transcription"` resolves to the provided transcription text
- [ ] `source = "keywords"` resolves to newline-joined keywords
- [ ] `source = "keywords"` with empty keywords list is skipped (no message produced)
- [ ] `file` inputs resolve to the file's contents
- [ ] `file` inputs with `~` paths expand correctly
- [ ] Missing file returns a clear error
- [ ] Inputs with no content source are skipped with a tracing warning
- [ ] `cargo check` passes
