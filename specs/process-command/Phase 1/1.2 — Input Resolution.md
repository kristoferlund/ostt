# 1.2 ‚Äî Input Resolution

## Problem

AI action inputs can reference dynamic sources (`"transcription"`, `"keywords"`) or external files (`file` field). These need to be resolved at runtime into concrete message strings before being sent to an LLM API.

## Objective

1. Create a function that takes a list of `ActionInput` entries and resolves them into a list of `ResolvedMessage` values (each containing an `InputRole` and a content string).
2. Handle all three content sources: literal `content`, `source` references, and `file` paths.

## Specification

### New module: `src/process/input.rs`

Also create `src/process/mod.rs` as the parent module.

```rust
/// A resolved message ready to send to an LLM.
pub struct ResolvedMessage {
    pub role: InputRole,
    pub content: String,
}

/// Resolves action inputs into concrete messages.
///
/// # Arguments
/// * `inputs` - The action's input configuration
/// * `transcription` - The transcribed text to substitute for `InputSource::Transcription`
/// * `keywords` - The user's keyword list to substitute for `InputSource::Keywords`
///
/// # Errors
/// - If a `file` path cannot be read
pub fn resolve_inputs(
    inputs: &[ActionInput],
    transcription: &str,
    keywords: &[String],
) -> anyhow::Result<Vec<ResolvedMessage>>
```

### Resolution logic

For each `ActionInput`, match on its `input_content` enum variant:

1. `InputContent::Literal { content }` ‚Äî use `content` as-is.
2. `InputContent::Source { source: InputSource::Transcription }` ‚Äî use the `transcription` argument.
3. `InputContent::Source { source: InputSource::Keywords }` ‚Äî join keywords with newlines. If keywords list is empty, skip this input entirely (don't send an empty message).
4. `InputContent::File { file }` ‚Äî read the file at `file`. Expand `~` to the user's home directory. Return error if file doesn't exist or can't be read.

All match arms are exhaustive ‚Äî both `InputContent` and `InputSource` are enums, so the compiler enforces that every case is handled. No fallback or "unknown source" handling needed.

### Module registration

Add `pub mod process;` to `src/lib.rs`.

## Acceptance Criteria

- [ ] Literal `content` inputs resolve to the exact string
- [ ] `source = "transcription"` resolves to the provided transcription text
- [ ] `source = "keywords"` resolves to newline-joined keywords

- [ ] `file` inputs resolve to the file's contents
- [ ] `file` inputs with `~` paths expand correctly
- [ ] Missing file returns a clear error
