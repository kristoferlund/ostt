# 5.1 — Process Subcommand

## Problem

There's no CLI entry point for processing. The `ostt process` subcommand needs to be added so users can post-process an existing transcription from history.

## Dependencies

- **Phase 3 (3.2)** — `process::execute_action` dispatcher
- **Phase 4 (4.1)** — `process::picker::show_action_picker` TUI

## Objective

1. Add the `process` subcommand (alias `p`) to the CLI.
2. Add a `get_transcription_by_index` method to `HistoryManager` for index-based lookup.
3. Implement the command handler that loads a transcription from history, optionally shows the action picker, executes the action, and outputs the result.

## Specification

### CLI definition in `src/app.rs`

Add to the `Commands` enum:

```rust
/// Post-process a transcription from history
///
/// Run a processing action on an existing transcription.
/// Shows the action picker if no --action is specified.
///
/// Examples:
///   ostt process                    # Process most recent, show picker
///   ostt process 3                  # Process #3, show picker  
///   ostt process -a clean           # Process most recent with "clean" action
///   ostt process 5 -a cmd -c        # Process #5 with "cmd", copy to clipboard
///   ostt process --list             # List configured actions
#[command(visible_alias = "p")]
Process {
    /// History index (1 = most recent, 2 = second most recent, etc.)
    #[arg(value_name = "N")]
    index: Option<usize>,

    /// Run action by ID, skip the action picker
    #[arg(short, long, value_name = "ID")]
    action: Option<String>,

    /// List all configured actions and exit
    #[arg(long)]
    list: bool,

    /// Copy result to clipboard instead of stdout (shadows global -c)
    #[arg(short, long)]
    clipboard: bool,

    /// Write result to file instead of stdout (shadows global -o)
    #[arg(short, long, value_name = "FILE")]
    output: Option<String>,
},
```

Note: The `-c` and `-o` flags shadow the global `Cli` flags. This is intentional and
consistent with how `Record`, `Retry`, and `Transcribe` define their own `-c`/`-o` flags.

### History lookup: `get_transcription_by_index` in `src/history/storage.rs`

Add a method to `HistoryManager`:

```rust
/// Gets the Nth most recent transcription (1-indexed).
///
/// # Arguments
/// * `index` - 1-based index (1 = most recent, 2 = second most recent, etc.)
///
/// # Errors
/// - If database connection fails
/// - If query execution fails
pub fn get_transcription_by_index(&mut self, index: usize) -> Result<Option<TranscriptionEntry>> {
    let connection = self.get_connection()?;
    let offset = index.saturating_sub(1);

    let mut statement = connection.prepare(
        "SELECT id, text, created_at FROM transcriptions ORDER BY created_at DESC LIMIT 1 OFFSET ?1",
    )?;

    let entry = statement
        .query_row(params![offset], |row| {
            // ... same parsing as get_transcription ...
        })
        .optional()?;

    Ok(entry)
}
```

### Command handler: `src/commands/process.rs`

```rust
pub async fn handle_process(
    index: Option<usize>,
    action_id: Option<String>,
    list: bool,
    clipboard: bool,
    output_file: Option<String>,
) -> Result<(), anyhow::Error>
```

#### `--list` mode

If `list` is true:
1. Load config.
2. If no actions are configured, print: `"No process actions configured. Add actions to ~/.config/ostt/ostt.toml"` and exit with code 0.
3. Otherwise, print each action as `{id} — {name}` (one per line) and exit.

#### Normal mode

1. Load config.
2. Validate that at least one action is configured. Error if none: `"No process actions configured. Add actions to ~/.config/ostt/ostt.toml"`.
3. Load transcription history via `HistoryManager`. Call `get_transcription_by_index(index.unwrap_or(1))`.
   - If history is empty or index is out of range, error: `"No transcription found at index {N}. Use 'ostt history' to see available transcriptions."`.
4. If `action_id` is given, look it up via `config.process.get_action(id)`. Error if not found: `"Unknown action '{id}'. Use 'ostt process --list' to see available actions."`.
5. If `action_id` is not given, show the action picker via `process::picker::show_action_picker`. If cancelled, exit cleanly (exit code 0, no output).
6. Load keywords via `KeywordsManager`.
7. Execute the action via `process::execute_action(&action, &transcription_text, &keywords)`.
8. Save the processed result to history via `history_manager.save_transcription(&result)`. The processed result is stored as a regular transcription entry — no special marking or schema distinction.
9. Output the result: file > clipboard > stdout (same priority as record/retry/transcribe).

### Routing in `src/app.rs`

Add the match arm in the `run()` function:

```rust
Some(Commands::Process { index, action, list, clipboard, output }) => {
    commands::handle_process(index, action, list, clipboard, output).await?;
}
```

### Module registration

Add `pub mod process;` to `src/commands/mod.rs` with `pub use process::handle_process;`.
