# 3.1 — Bash Action Executor

## Problem

Bash processing actions need to pipe the transcribed text into a shell command's stdin and capture the command's stdout as the processed result.

## Objective

1. Implement a function that executes a bash command, pipes input to stdin, and returns stdout.
2. Handle errors (command not found, non-zero exit, stderr output).

## Specification

### New module: `src/process/bash.rs`

```rust
/// Executes a bash command with the given input piped to stdin.
///
/// Returns the command's stdout as a trimmed string.
///
/// # Arguments
/// * `command` - The shell command to execute (passed to `sh -c`)
/// * `input` - Text to pipe into the command's stdin
///
/// # Errors
/// - If the command cannot be spawned
/// - If stdin cannot be written
/// - If the command exits with non-zero status
/// - If stdout cannot be read
pub async fn execute_bash_action(command: &str, input: &str) -> anyhow::Result<String>
```

### Implementation details

1. Spawn `sh -c <command>` with stdin piped and stdout captured.
2. Write `input` to the child's stdin, then close it.
3. Wait for the child to complete.
4. If exit status is non-zero, include stderr in the error message.
5. Return trimmed stdout.
6. Use `tokio::process::Command` for async compatibility.

### Timeout

Add a 30-second timeout. If the command takes longer, kill it and return an error.

### Error messages

- Command not found: "Command failed to start: {error}. Make sure the command is installed."
- Non-zero exit: "Command exited with status {code}:\n{stderr}"
- Timeout: "Command timed out after 30 seconds"

## Acceptance Criteria

- [ ] `execute_bash_action("tr '[:lower:]' '[:upper:]'", "hello")` returns `"HELLO"`
- [ ] `execute_bash_action("cat", "pass through")` returns `"pass through"`
- [ ] A command that exits with non-zero status returns an error containing stderr
- [ ] A non-existent command returns a clear error
- [ ] `cargo check` passes
