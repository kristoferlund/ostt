# 3.2 — Action Dispatcher

## Problem

There's no unified function to execute a processing action. The caller would need to check the action type and dispatch to either the bash executor or the chat completions client. This logic should be centralized.

## Objective

1. Create a single `execute_action` function that takes a `ProcessAction` and transcription text, resolves inputs, and dispatches to the correct executor.
2. This is the primary API for the rest of the codebase to invoke processing.

## Specification

### New module: `src/process/execute.rs`

```rust
/// Executes a processing action on the given transcription text.
///
/// For bash actions: pipes transcription to the command's stdin.
/// For AI actions: resolves inputs, sends to chat completions API, returns response.
///
/// # Arguments
/// * `action` - The action configuration to execute
/// * `transcription` - The raw transcription text
/// * `keywords` - The user's keyword list (for AI actions that reference keywords)
///
/// # Errors
/// - If the action type is bash and the command fails
/// - If the action type is AI and the API call fails
/// - If input resolution fails (missing file, etc.)
/// - If the action is misconfigured (no command for bash, no model for AI)
pub async fn execute_action(
    action: &ProcessAction,
    transcription: &str,
    keywords: &[String],
) -> anyhow::Result<String>
```

### Dispatch logic

```
match action.action_type {
    ActionType::Bash => {
        let command = action.command.as_ref()
            .ok_or_else(|| anyhow!("Bash action '{}' has no command", action.id))?;
        bash::execute_bash_action(command, transcription).await
    }
    ActionType::Ai => {
        let model_spec = action.model.as_ref()
            .ok_or_else(|| anyhow!("AI action '{}' has no model", action.id))?;
        let (provider_id, model_name) = chat::parse_model_spec(model_spec)?;
        let api_key = config::get_api_key(&provider_id)?
            .ok_or_else(|| anyhow!("No API key for provider '{}'. Run 'ostt auth'.", provider_id))?;
        let messages = input::resolve_inputs(&action.inputs, transcription, keywords)?;
        // ... build ChatConfig, call chat_completions
    }
}
```
