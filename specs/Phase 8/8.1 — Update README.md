### Changes to README.md

#### Features section

Add a bullet to the features list:

```markdown
- **Post-processing pipeline** - Chain AI or bash actions after transcription to clean up text, translate, generate commands, and more
```

#### Commands section

Add the new commands and flags to the command reference block:

```bash
ostt process         # Post-process most recent transcription (show action picker)
ostt process 3       # Post-process transcription #3 from history
ostt process -a clean   # Run the "clean" action on most recent transcription
ostt process --list  # List configured processing actions
ostt -p              # Record, transcribe, then process (show picker)
ostt -p clean -c     # Record, transcribe, run "clean" action, copy to clipboard
ostt record -p clean # Same as above (explicit record subcommand)
ostt transcribe f -p clean  # Transcribe file and process
ostt retry -p clean  # Retry transcription and process
```

Add `p` (process) to the command aliases line.

#### Configuration section

Add a processing configuration example after the existing `[providers.*]` examples:

```toml
# Processing actions
[[process.actions]]
id = "clean"
name = "Clean up text"
type = "ai"
tool = "claude-code"
model = "sonnet"
inputs = [
    { role = "system", content = "Clean up the following transcribed text. Fix grammar, remove filler words, and improve clarity. Return only the cleaned text." },
    { role = "user", source = "transcription" },
]

[[process.actions]]
id = "cmd"
name = "Generate CLI command"
type = "bash"
command = "tr '[:lower:]' '[:upper:]'"
```

#### AI Tool Configuration Examples

Add a dedicated section (or sub-section within Configuration) showing how to configure
AI processing actions for each of the four supported tools. Each example should show a
complete action block with the correct `tool` value, a realistic `model` value, and any
relevant notes about tool-specific behavior.

The four tools to document:

1. **Claude Code** (`tool = "claude-code"`) — model examples: `"sonnet"`, `"opus"`.
   Note: OSTT automatically passes `--allowedTools ""`, `--strict-mcp-config`, and
   `--no-session-persistence` to ensure Claude Code operates as a pure text processor
   (no tool use, no MCP servers, no session persistence).

2. **OpenCode** (`tool = "open-code"`) — model examples: `"anthropic/claude-sonnet-4-6"`,
   `"opencode/kimi-k2.5"`. Note: uses `provider/model` format. The system prompt is
   included in stdin (not as a CLI flag).

3. **Gemini CLI** (`tool = "gemini-cli"`) — model examples: `"gemini-2.5-flash"`,
   `"gemini-2.5-pro"`.

4. **Codex CLI** (`tool = "codex-cli"`) — model examples: `"o3"`, `"o4-mini"`.
   Note: requires OpenAI API key auth (`codex login --with-api-key`), not ChatGPT
   account auth.

Each example should use the same "clean up text" action so the reader can compare
the tool configuration differences side-by-side.

Also note that `tool_binary` and `tool_args` are optional overrides for advanced use
(custom binary path, extra CLI flags).

#### Hyprland Hotkey Integration

Add a section documenting how to set up hotkey-driven workflows on Hyprland using the
`ostt-float` script (provided in `environments/hyprland/`). Cover:

1. **Basic setup** — how to install `ostt-float.sh` and `alacritty-float.toml`, where
   they go (`~/.local/bin/` and `~/.config/ostt/`).

2. **Plain recording hotkey** — bind a key to `ostt-float -c` for record → transcribe →
   clipboard. Explain the two-press workflow (first press starts recording, second press
   triggers transcription).

3. **Processing hotkey** — bind a second key to `ostt-float -p clean -c` for
   record → transcribe → clean → clipboard. Explain that `-p <action>` chains processing
   after transcription and that any ostt arguments work since the float script passes
   them through.

4. **Example bindings.conf snippet:**
   ```
   bindd = SUPER, R, ostt, exec, bash ~/.local/bin/ostt-float
   bindd = SUPER SHIFT, R, ostt clean, exec, bash ~/.local/bin/ostt-float -p clean -c
   ```

5. **Two-window workflow with action picker** — for users who want the interactive picker
   in a larger window. Explain the two-invocation model:
   ```sh
   ostt-float -c && ostt-float -- process -c
   ```
   Note that the window manager can apply different rules to each window based on title.

6. **Custom binary override** — mention `OSTT_BIN` env var for pointing at a dev build
   or alternative installation path.
