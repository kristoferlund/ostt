# 6.1 — History Schema for Processed Results

## Problem

The current history system saves a single text string per transcription. With processing, each run can produce two artifacts: the raw transcription and the processed result. Both should be preserved in history. The `ostt process` command also needs to look up transcriptions by index (most recent first) to operate on them.

## Objective

1. Extend the SQLite schema to store both raw transcriptions and processed results.
2. Update the history manager API to support saving and querying processed results.
3. Ensure `ostt process N` can look up the Nth most recent transcription.

## Specification

### Schema change

Drop and recreate the `transcriptions` table (no migration needed — 0 real users per AGENTS.md):

```sql
CREATE TABLE IF NOT EXISTS transcriptions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    text TEXT NOT NULL,
    source TEXT NOT NULL DEFAULT 'transcription',
    action_id TEXT,
    parent_id INTEGER,
    created_at TEXT NOT NULL,
    FOREIGN KEY (parent_id) REFERENCES transcriptions(id)
)
```

New columns:
- `source` — either `"transcription"` (raw STT result) or `"processed"` (action output)
- `action_id` — the action ID that produced this entry (NULL for raw transcriptions)
- `parent_id` — for processed results, the ID of the raw transcription it was derived from (NULL for raw transcriptions)

### Updated TranscriptionEntry

```rust
pub struct TranscriptionEntry {
    pub id: i64,
    pub text: String,
    pub source: String,
    pub action_id: Option<String>,
    pub parent_id: Option<i64>,
    pub created_at: DateTime<Local>,
}
```

### New HistoryManager methods

```rust
/// Saves a raw transcription to history. Returns the entry ID.
pub fn save_transcription(&mut self, text: &str) -> Result<i64>

/// Saves a processed result to history.
/// Links it to the parent transcription via parent_id.
pub fn save_processed_result(
    &mut self,
    text: &str,
    action_id: &str,
    parent_id: i64,
) -> Result<i64>

/// Gets the Nth most recent transcription (1-indexed).
/// Only returns raw transcriptions (source = 'transcription'), not processed results.
/// This is used by `ostt process N`.
pub fn get_transcription_by_index(&mut self, index: usize) -> Result<Option<TranscriptionEntry>>

/// Gets all history entries (both raw and processed) for display.
pub fn get_all_entries(&mut self) -> Result<Vec<TranscriptionEntry>>
```

### Behavioral changes

- `save_transcription` now returns the entry ID (currently returns `()`). This is needed so that the calling code can pass the ID to `save_processed_result` as `parent_id`.
- `get_all_transcriptions` is renamed to `get_all_entries` and returns all entries (both raw and processed).
- `get_transcription_by_index` only searches `source = 'transcription'` entries, ordered by `created_at DESC`. This ensures `ostt process 1` targets the most recent raw transcription, not a processed result.

### Impact on existing callers

- `handle_record`, `handle_retry`, `handle_transcribe` — update to capture the returned ID from `save_transcription` and pass it to `save_processed_result` when processing.
- `handle_history` — update to use `get_all_entries`. Display both raw and processed entries. Add a visual indicator for processed entries (e.g., prefix with the action name).
