# 5.1 — Process Subcommand

## Problem

There's no CLI entry point for processing. The `ostt process` subcommand needs to be added so users can post-process an existing transcription from history.

## Objective

1. Add the `process` subcommand (alias `p`) to the CLI.
2. Implement the command handler that loads a transcription from history, optionally shows the action picker, executes the action, and outputs the result.

## Specification

### CLI definition in `src/app.rs`

Add to the `Commands` enum:

```rust
/// Post-process a transcription from history
///
/// Run a processing action on an existing transcription.
/// Shows the action picker if no --action is specified.
///
/// Examples:
///   ostt process                    # Process most recent, show picker
///   ostt process 3                  # Process #3, show picker  
///   ostt process -a clean           # Process most recent with "clean" action
///   ostt process 5 -a cmd -c        # Process #5 with "cmd", copy to clipboard
///   ostt process --list             # List configured actions
#[command(visible_alias = "p")]
Process {
    /// History index (1 = most recent, 2 = second most recent, etc.)
    #[arg(value_name = "N")]
    index: Option<usize>,

    /// Run action by ID, skip the action picker
    #[arg(short, long, value_name = "ID")]
    action: Option<String>,

    /// List all configured actions and exit
    #[arg(long)]
    list: bool,

    /// Copy result to clipboard instead of stdout
    #[arg(short, long)]
    clipboard: bool,

    /// Write result to file instead of stdout
    #[arg(short, long, value_name = "FILE")]
    output: Option<String>,
},
```

### Command handler: `src/commands/process.rs`

```rust
pub async fn handle_process(
    index: Option<usize>,
    action_id: Option<String>,
    list: bool,
    clipboard: bool,
    output_file: Option<String>,
) -> Result<(), anyhow::Error>
```

#### `--list` mode

If `list` is true:
1. Load config
2. Print each action as `{id} — {name}` (one per line)
3. Exit

#### Normal mode

1. Load config and validate actions exist.
2. Load transcription history. Get the entry at `index` (default 1, most recent).
3. If `action_id` is given, look it up in config. Error if not found.
4. If `action_id` is not given, show the action picker. If cancelled, exit cleanly.
5. Load keywords.
6. Execute the action via `process::execute_action`.
7. Save the processed result to history (see Phase 6 spec for details — for now, save as a regular transcription entry with a `[Processed]` prefix).
8. Output the result: file > clipboard > stdout (same priority as record/retry/transcribe).

### Routing in `src/app.rs`

Add the match arm in the `run()` function:

```rust
Some(Commands::Process { index, action, list, clipboard, output }) => {
    commands::handle_process(index, action, list, clipboard, output).await?;
}
```

### Module registration

Add `pub mod process;` to `src/commands/mod.rs` with the appropriate `pub use`.
